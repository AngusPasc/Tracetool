Although most development projects will standardize on one of these variations, the Event Sources API sample demonstrates the different ways you can invoke the event source object. 

Features 

·        A class-static event source instance.

·        Validation of enabled event types prior to raising them.

·        Events within the same method call raised through multiple event sources.

Classes and Keywords

The sample uses the following classes, methods, and properties from the Microsoft.EnterpriseInstrumentation and Microsoft.EnterpriseInstrumentation.Schema namespaces:

·        EventSource.Application

·        EventSource.IsEnabledForType

·        EventSource.Raise

·        TraceMessageEvent

·        ProjectInstaller

Installing and Running the Event Sources Sample

To build, install, and run the sample
  1. Open the "Event Sources" project in Visual Studio .NET. Click one of the Event Source API Solutions links above for the development language (C# or VB .NET) that you wish to use. In the File Download dialog, select Open this file from its current location, and then click OK.

    The Event Source API project opens in the Code Editor.
  2. On the Build menu, click Build Solution.
  3. Using the Visual Studio .NET command prompt, access the build output directory, for example "Samples\Event Sources\cs\bin\Debug". Run InstallUtil.exe from the command line with EventSourceAPI.exe as a parameter. Type the following:

    installutil EventSourceAPI.exe
  4. Edit the resulting EnterpriseInstrumentation.config to add event filter bindings for desired event categories and event sinks. For example, you can replace the defaultSoftwareElementFilter definition with the following:

    <filter name="defaultSoftwareElementFilter" description=" A default filter for the SoftwareElement event sources.">
       <eventCategoryRef name="All Events">
          <eventSinkRef name="wmiSink" />
          <eventSinkRef name="traceSink" />
          <eventSinkRef name="logSink" />
       </eventCategoryRef>
    </filter>
  5. On the Visual Studio Debug menu, click Start or Start Without Debugging.
