The Simple Instrumentation sample provides a "Hello World"-style introduction to instrumentation.

Features

  • Basic one-line invocation of TraceMessageEvent.
  • Fires events through the implicit Application event source.

Classes and Keywords

The sample uses the following classes and properties from the Microsoft.EnterpriseInstrumentation and Microsoft.EnterpriseInstrumentation.Schema namespaces:

  • EventSource.Application
  • ProjectInstaller
  • TraceMessageEvent

Installing and Running the Simple Instrumentation Sample

To build, install, and run the sample
  1. Open the "Simple Instrumentation" project in Visual Studio .NET. Click one of the Simple Instrumentation Solutions links above for the development language (C# or VB .NET) that you wish to use. In the File Download dialog, select Open this file from its current location, and then click OK.

    The Simple Instrumentation project opens in the Code Editor.
  2. On the Build menu, click Build Solution.
  3. Using the Visual Studio .NET command prompt, access the build output directory, for example "Samples\Simple Instrumentation\cs\bin\Debug". Run InstallUtil.exe from the command line with SimpleInstrumentation.exe as a parameter. Type the following:

    installutil SimpleInstrumentation.exe
  4. Edit the resulting EnterpriseInstrumentation.config to add event filter bindings for desired event categories and event sinks. For example, you can replace the defaultSoftwareElementFilter definition with the following:

    <filter name="defaultSoftwareElementFilter" description=" A default filter for the SoftwareElement event sources.">
       <eventCategoryRef name="All Events">
          <eventSinkRef name="wmiSink" />
          <eventSinkRef name="traceSink" />
          <eventSinkRef name="logSink" />
       </eventCategoryRef>
    </filter>
  5. On the Visual Studio Debug menu, click Start or Start Without Debugging.
