Although most development projects will standardize on one of these variations, the Request Tracing sample demonstrates the different ways you can implement a request trace. 

Features 

  • Explicit request tracing using new RequestTrace object, and static RequestEventSource.
  • Demonstration of single event raised simultaneously to multiple event sources.
  • Side-by-side comparison of using vs. try-finally paradigms.

Classes and Keywords

The sample uses the following classes from the Microsoft.EnterpriseInstrumentation, Microsoft.EnterpriseInstrumentation.RequestTracing, and Microsoft.EnterpriseInstrumentation.Schema namespaces:

  • ProjectInstaller
  • RequestEventSource
  • RequestTrace
  • TraceMessageEvent

Installing and Running the Request Tracing Sample

To build, install, and run the sample
  1. Open the "Request Tracing" project in Visual Studio .NET. Click one of the Request Tracing Solutions links above for the development language (C# or VB .NET) that you wish to use. In the File Download dialog, select Open this file from its current location, and then click OK.

    The Request Tracing project opens in the Code Editor.
  2. On the Build menu, click Build Solution.
  3. Using the Visual Studio .NET command prompt, access the build output directory, for example "Samples\Request Tracing\cs\bin\Debug". Run InstallUtil.exe from the command line with RequestTracing.exe as a parameter. Type the following:

    installutil RequestTracing.exe
  4. Edit the resulting EnterpriseInstrumentation.config to add event filter bindings for desired event categories and event sinks. Because this sample makes use of different event source types, it is easier to distinguish message types if defaultSoftwareElementFilter and defaultRequestFilter use separate event sinks. For example, you can replace the default filter definitions with the following:

    <filter name="defaultSoftwareElementFilter" description=" A default filter for the SoftwareElement event sources.">
       <eventCategoryRef name="All Events">
          <eventSinkRef name="traceSink" />
       </eventCategoryRef>
    </filter>
    <filter name="defaultRequestFilter" description=" A default filter for Request event sources.">
       <eventCategoryRef name="All Events">
          <eventSinkRef name="wmiSink" />
          <eventSinkRef name="logSink" />
       </eventCategoryRef>
    </filter>
  5. On the Visual Studio Debug menu, click Start or Start Without Debugging.
